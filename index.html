0:49:48 A Block in a Block Chain:

  • Genesis block is first block in the chain. (And has zeros for Previous, see next line)
  • The blocks in a chain have a 'Previous' input window added which shows the hash of the previous block.
  • Attempting to change data in a block invalidates all subsequent blocks.
  • Re-mining will re-validate the block but will show a different hash and blocks on other computers (Peer views) will show the original hash indicating a change!
  • Out of step hashes show a change in the block from the original. The offending chain of blocks will be excluded.
  • By adding a Coinbase input window for transactions, previous blocks will indicate funds available and provenance.

How it goes 'in actuality'

  • Node - a single instance of a decentralised network which anyone can join.
  • Ethereum can carry out computations for mining - Concensus - Proof of stake, Proof of work,
  • Consensus protocol has 1- Chain selection, 2- Sybil resistance mechanism
  • To 1:13:23 SybilControl mechanism consists of a distributed protocol to allow nodes to collectively verify the computational work of other nodes, and mechanisms to prevent the malicious influence of misbehaving nodes that do not perform the computational work.

Proof of Work

  • PoW verifies the transaction node and the transaction (the Nonce) and is an SRM. The length of the mining algorithm can vary in length which affects the complexity of the computation (Block Time). Each malicious node has to be mined and will (can) be eliminated.
  • To find the real node/transaction we use a Chain Selection Rule called Chain, or Nakamoto, Consensus which combines PoW and longest chain.
  • The first node to solve the Nonce earns the Transaction fee (the Gas) and the Block award
  • Attacks. Sybil or 51% (can happen on Etherum Classic but not Etherum?)
  • Longest chain rule

Proof of Stake

  • PoStake - gas paid to validators, discussed later
  • PoW uses lots of electricity. PoStake is the most popular alternative verification and many newcomers incorporate PoS including in ETH 2.0
  • PoS miners are called validators, they are chosen randomly they and all remaining validatores put up collateral as non-Sybil attack garauntee
  • Randomness in a determinist system which by def does not have random numbers, can be generated through EG in ETH2 by RANDAO, a decentralised, autonomous organisation using a smart contract in the blockchain.
  • PoS pros - civil resistance mechanism, reveals block author, less computation therefore power usage through PoS decentralisation. Further use will determine how much decentralisation will PoS need.

Scaleability

  • A high numbers of users results in higher 'gas' charges.
  • Can be alleviated by 'sharding' - a block chain of block chains which co-ordinates the higher number of users by Layers.
  • Layer 1 (baselayer) - Bitcoin, Ethereum, Avalanche. Layer 2 is an application added on top of L1. EG Chainlink, Arbitrum, Optimism. A & O are known as 'roll-ups' which r/up into Layer 1 using Layer 1 security and act like a sharded chain.
